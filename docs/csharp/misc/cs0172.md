# <a name="compiler-error-cs0172"></a><span data-ttu-id="3d1f0-103">编译器错误 CS0172</span><span class="sxs-lookup"><span data-stu-id="3d1f0-103">Compiler Error CS0172</span></span>
<span data-ttu-id="3d1f0-104">无法确定条件表达式的类型，因为“type1”和“type2”可相互隐式转换</span><span class="sxs-lookup"><span data-stu-id="3d1f0-104">Type of conditional expression cannot be determined because 'type1' and 'type2' implicitly convert to one another</span></span>  
  
<span data-ttu-id="3d1f0-105">在条件语句中，必须能够隐式转换标记两侧的类型 `:` 。</span><span class="sxs-lookup"><span data-stu-id="3d1f0-105">In a conditional statement, you must be able to implicitly convert the types on either side of the `:` token.</span></span> <span data-ttu-id="3d1f0-106">此外，不能有相互的隐式转换;只需要一个转换。</span><span class="sxs-lookup"><span data-stu-id="3d1f0-106">Also, there cannot be mutual implicit conversions; you only need one conversion.</span></span>
  
<span data-ttu-id="3d1f0-107">下面的示例生成 CS0172：</span><span class="sxs-lookup"><span data-stu-id="3d1f0-107">The following sample generates CS0172:</span></span>
  
```csharp  
// CS0172.cs  
public class Square  
{  
   public class Circle  
   {  
      public static implicit operator Circle(Square aa)  
      {  
         return null;  
      }  
  
      public static implicit operator Square(Circle aa)  
      // using explicit resolves this error  
      // public static explicit operator Square(Circle aa)  
      {  
         return null;  
      }  
   }  
  
   public static void Main()  
   {  
      Circle aa = new Circle();  
      Square ii = new Square();  
      object o = (1 == 1) ? aa : ii;   // CS0172  
      // the following cast would resolve this error  
      // (1 == 1) ? aa : (Circle)ii;  
   }  
}  
```
