## <a name="mapping-error-codes-to-exceptions"></a><span data-ttu-id="32d00-118">将错误代码映射到异常</span><span class="sxs-lookup"><span data-stu-id="32d00-118">Mapping error codes to exceptions</span></span>

<span data-ttu-id="32d00-119">由于文件系统为操作系统资源，.NET Core 和 .NET Framework 中的 I/O 方法将包装对基础操作系统的调用。</span><span class="sxs-lookup"><span data-stu-id="32d00-119">Because the file system is an operating system resource, I/O methods in both .NET Core and .NET Framework wrap calls to the underlying operating system.</span></span> <span data-ttu-id="32d00-120">当由操作系统执行的代码出现 I/O 错误时，操作系统将对 .NET I/O 方法返回错误信息。</span><span class="sxs-lookup"><span data-stu-id="32d00-120">When an I/O error occurs in code executed by the operating system, the operating system returns error information to the .NET I/O method.</span></span> <span data-ttu-id="32d00-121">然后，该方法会将错误信息（通常采用错误代码形式）转换为 .NET 异常类型。</span><span class="sxs-lookup"><span data-stu-id="32d00-121">The method then translates the error information, typically in the form of an error code, into a .NET exception type.</span></span> <span data-ttu-id="32d00-122">大多数情况下，可以通过直接将错误代码转换为其相应异常类型来完成此操作；它不基于方法调用的上下文执行任何特殊的错误映射。</span><span class="sxs-lookup"><span data-stu-id="32d00-122">In most cases, it does this by directly translating the error code into its corresponding exception type; it does not perform any special mapping of the error based on the context of the method call.</span></span>

<span data-ttu-id="32d00-123">例如，在 Windows 操作系统，返回 `ERROR_FILE_NOT_FOUND`（或 0x02）错误代码的方法调用会映射到 <xref:System.IO.FileNotFoundException>，`ERROR_PATH_NOT_FOUND` 错误代码（或 0x03）则映射到 <xref:System.IO.DirectoryNotFoundException>。</span><span class="sxs-lookup"><span data-stu-id="32d00-123">For example, on the Windows operating system, a method call that returns an error code of `ERROR_FILE_NOT_FOUND` (or 0x02) maps to a <xref:System.IO.FileNotFoundException>, and an error code of `ERROR_PATH_NOT_FOUND` (or 0x03) maps to a <xref:System.IO.DirectoryNotFoundException>.</span></span>

<span data-ttu-id="32d00-124">但是，操作系统返回特定错误代码的精确条件通常未记录或记录不当。</span><span class="sxs-lookup"><span data-stu-id="32d00-124">However, the precise conditions under which the operating system returns particular error codes is often undocumented or poorly documented.</span></span> <span data-ttu-id="32d00-125">因此，会出现意外异常。</span><span class="sxs-lookup"><span data-stu-id="32d00-125">As a result, unexpected exceptions can occur.</span></span> <span data-ttu-id="32d00-126">例如，因为使用的是目录而不是文件，可以预料到向 <xref:System.IO.DirectoryInfo.%23ctor%2A> 构造函数提供无效目录路径将引发 <xref:System.IO.DirectoryNotFoundException>。</span><span class="sxs-lookup"><span data-stu-id="32d00-126">For example, because you are working with a directory rather than a file, you would expect that providing an invalid directory path to the <xref:System.IO.DirectoryInfo.%23ctor%2A> constructor throws a <xref:System.IO.DirectoryNotFoundException>.</span></span> <span data-ttu-id="32d00-127">但是，它也可能引发 <xref:System.IO.FileNotFoundException>。</span><span class="sxs-lookup"><span data-stu-id="32d00-127">However, it may also throw a <xref:System.IO.FileNotFoundException>.</span></span>

## <a name="handling-ioexception"></a><span data-ttu-id="32d00-154">处理 IOException</span><span class="sxs-lookup"><span data-stu-id="32d00-154">Handling IOException</span></span>

<span data-ttu-id="32d00-155">作为 <xref:System.IO> 命名空间中异常的基类，当任何错误代码未映射到预定义的异常类型时，也将引发 <xref:System.IO.IOException>。</span><span class="sxs-lookup"><span data-stu-id="32d00-155">As the base class for exceptions in the <xref:System.IO> namespace, <xref:System.IO.IOException> is also thrown for any error code that does not map to a predefined exception type.</span></span> <span data-ttu-id="32d00-156">这意味着异常可以由任何 I/O 操作引发。</span><span class="sxs-lookup"><span data-stu-id="32d00-156">This means that it can be thrown by any I/O operation.</span></span>
